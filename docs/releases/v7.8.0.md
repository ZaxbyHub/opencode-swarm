# v7.8.0

## What changed

### Feature: Full-Auto v2 — autonomy control plane with deterministic policy + critic oversight

Full-Auto now provides real autonomy without compromising safety. The legacy
v1 reactive-intercept behavior is preserved as a compatibility fallback.

#### New surface

- `full_auto.mode`: `"assisted" | "supervised" | "strict"` (default
  `"supervised"`) — controls how aggressively the critic is invoked.
- `full_auto.fail_closed` (default `true`) — pause the run when the
  oversight infrastructure is unavailable instead of silently allowing.
- `full_auto.permission_policy.{enabled,trusted_roots,trusted_domains,protected_paths,allow_defaults}`
  — deterministic allow/deny/escalate classifier for every tool call.
- `full_auto.denials.{max_consecutive,max_total,on_limit}` — durable
  denial accounting; the run pauses (or terminates) when limits are
  reached.
- `full_auto.oversight.{on_plan_change,on_task_completion,on_phase_boundary,on_high_risk_action,on_subagent_return_warning,every_tool_calls,every_architect_turns,every_minutes}`
  — periodic and risk-triggered critic cadence.

#### New runtime artifacts

- `.swarm/full-auto-state.json` — durable per-session run state
  (`status: "running" | "paused" | "terminated"`, denial counters,
  oversight cadence, phase/task identity).
- `.swarm/evidence/{phase}/full-auto-{seq}.json` — per-phase Full-Auto
  oversight evidence consumed by `phase_complete`.
- `events.jsonl` now records `full_auto_oversight`,
  `full_auto_input_warning`, and `full_auto_subagent_warning` records in
  addition to the legacy `auto_oversight` records.

#### New hooks (`src/hooks/`)

- `full-auto-permission.ts` — `tool.execute.before` permission decision
  after guardrails / scope-guard / delegation-gate. Throws structured
  denials; dispatches the critic on escalation; pauses on fail-closed
  paths. Wired via raw `await` (NOT `safeHook` / `composeHandlers`) so
  throws propagate to the OpenCode host as tool rejections.
- `full-auto-input-probe.ts` — `tool.execute.after` prompt-injection
  scanner (instruction overrides, system role overrides, credential
  requests, exfiltration patterns, guardrail disable, curl-pipe-shell);
  pairs with the permission hook so risky follow-ups escalate.
- `full-auto-delegation.ts` — Task delegation outbound check (registered
  canonical role, declared coder scope, protected paths in prompt) and
  return check (skipped tests, scope changes, external instructions,
  missing evidence, out-of-scope file generation).

#### New helpers (`src/full-auto/`)

- `policy.ts` — pure deterministic classifier (`classifyFullAutoToolAction`,
  `classifyPathRisk`, `classifyCommandRisk`, `isProtectedPath`,
  `isReadOnlyTool`, `isWriteLikeTool`, `isSubagentDelegation`,
  `buildStructuredDenial`).
- `oversight.ts` — shared critic dispatcher (`dispatchFullAutoOversight`,
  `parseFullAutoCriticResponse`, `writeFullAutoOversightEvent`,
  `writeFullAutoOversightEvidence`).
- `state.ts` — durable Full-Auto run state with atomic
  tmp→fsync→rename writes, .bak fallback, cross-process `lockSync`,
  monotonic oversight sequence counter, and a `stateUnreadable` sentinel
  that fails-closed when corrupt/unrecoverable.
- `phase-approval.ts` — phase-completion approval gate consumed by
  `phase_complete` (Gate 7, after Final Council).
- `cadence.ts` — periodic/risk cadence evaluation + non-blocking
  dispatcher.
- `input-probe.ts` — pure prompt-injection probe.

#### Behavior changes

- `phase_complete` now blocks when Full-Auto v2 is active and there is
  no fresh APPROVED `full_auto_oversight` evidence record for the phase.
  Turbo does NOT bypass this gate. Runs as Gate 7 after the Final
  Council gate added in v7.7.0.
- The reactive intercept (`src/hooks/full-auto-intercept.ts`) continues
  to fire on architect text patterns and writes legacy `auto_oversight`
  events. When a durable Full-Auto run exists, the same verdict is
  mirrored into the v2 `full_auto_oversight` event/evidence pipeline so
  phase approval and durable state stay coherent.
- `AGENT_TOOL_MAP.critic_oversight` expanded from 6 to 20 read-only
  verification tools (`diff`, `evidence_check`, `check_gate_status`,
  `completion_verify`, `get_approved_plan`, `req_coverage`,
  `test_impact`, `pkg_audit`, `secretscan`, `sast_scan`, `repo_map`,
  etc.). No write/edit/patch tools were added.
- `/swarm full-auto on` now creates the durable v2 run-state record
  (writing it BEFORE flipping the legacy `session.fullAutoMode` flag —
  fail-closed) and reports the resolved mode, denial limits, and
  `fail_closed` status. `/swarm full-auto off` pauses the run state.

#### Canonical role extraction (replaces hardcoded swarm-prefix list)

Swarm IDs are user-defined arbitrary strings — the plugin must not enumerate
them. The previous `KNOWN_SWARM_PREFIXES` list (`local`, `mega`, `paid`,
`cloud`, `enterprise`, `default`, `custom`, `team`, `project`, `swarm`,
`synthetic`) has been removed. Canonical roles are now extracted by
longest-suffix matching against `ALL_AGENT_NAMES`:

- `getCanonicalAgentRole(name, generatedAgentNames?)` — pure role
  extraction; `arbitrary_critic_oversight` resolves to `critic_oversight`,
  never `critic` or `oversight`.
- `isKnownCanonicalRole(role)` — type guard against `ALL_AGENT_NAMES`.
- `resolveGeneratedAgentRole(name, registry)` — strict variant that
  only infers a role for names actually generated by the swarm registry;
  protects against treating user-supplied prose like `not_an_architect`
  as a role.

`stripKnownSwarmPrefix` is preserved as a backward-compatible alias
delegating to `getCanonicalAgentRole` so all existing call sites and
tests continue to work. Generated agent names (e.g. `bananaSwarm_coder`,
`acme-prod_reviewer`, `customer123_critic_oversight`) are preserved
EXACTLY for OpenCode dispatch — only role-filtering / policy /
guardrails consult the canonical role.

#### Hook composition fail-closed contract

Added `composeBlockingHandlers` to `src/hooks/utils.ts` as a documented
helper for fail-closed hook composition (companion to `composeHandlers`,
which wraps every handler in `safeHook` and is appropriate for advisory
chains only). The actual `tool.execute.before` registration in
`src/index.ts` continues to use raw `await` calls for each fail-closed
hook, with explicit `FAIL-CLOSED CHAIN` comments and a regression test
(`tests/unit/hooks/hook-composition.test.ts`) that asserts none of the
Full-Auto v2 / guardrail / scope-guard / delegation-gate hooks are
wrapped in `safeHook` in either `src/index.ts` or the built
`dist/index.js`.

### Why

The pre-v2 Full-Auto provided session-scoped autonomy but had no
deterministic action governance, no denial accounting, and no
fail-closed semantics. Decisions depended on architect-text patterns
and a single critic dispatch per escalation, with no enforcement at
the tool boundary. Full-Auto v2 adds an Anthropic-style autonomy
control plane: a deterministic classifier handles obvious cases, the
read-only `critic_oversight` agent verifies ambiguous cases with
positive evidence, and durable state ensures pause/terminate decisions
survive process restarts. Denials become a first-class signal the
agent can recover from; phase approval becomes an auditable evidence
record.

### Migration steps

No migration required for existing configs. v1 fields
(`max_interactions_per_phase`, `deadlock_threshold`, `escalation_mode`,
`critic_model`) continue to work unchanged. Optional v2 keys can be
added when desired. To enable v2 features:

```jsonc
{
  "full_auto": {
    "enabled": true,
    "mode": "supervised",
    "fail_closed": true,
    "denials": { "max_consecutive": 3, "max_total": 20, "on_limit": "pause" },
    "oversight": { "on_phase_boundary": true, "on_high_risk_action": true }
  }
}
```

To recover from a paused run, inspect `.swarm/full-auto-state.json` for
`pauseReason` / denial history, address the underlying cause, then run
`/swarm full-auto on` to start a fresh running record.
